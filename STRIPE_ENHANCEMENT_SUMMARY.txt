WEBHOOK FLOW
================================================================================

INCOMING EVENT: checkout.session.completed
  ↓
  1. Verify signature (STRIPE_WEBHOOK_SECRET)
  ↓
  2. Log event to stripe_webhook_logs (for audit)
  ↓
  3. Check idempotency (stripe_session_id unique)
  ↓
  4. Validate metadata (userId, credits, packId)
  ↓
  5. Lock user profile row (prevent race conditions)
  ↓
  6. Check sufficient balance exists
  ↓
  7. Update credit_balance atomically
  ↓
  8. Insert credit_transactions with:
     - payment_status = 'succeeded'
     - invoice_url from Stripe
     - stripe_session_id for idempotency
  ↓
  9. Send confirmation email (async):
     - User email
     - Credits added
     - Invoice download link
     - Transaction ID
  ↓
  10. Log email notification to payment_notifications
  ↓
  11. Return 200 OK (tells Stripe to stop retrying)
  ↓
SUCCESS: Credits in account, invoice saved, email sent

================================================================================
FAILURE HANDLING
================================================================================

INCOMING EVENT: payment_intent.payment_failed
  ↓
  1. Log event to stripe_webhook_logs
  ↓
  2. Find transaction by stripe_payment_id
  ↓
  3. Update payment_status = 'failed'
  ↓
  4. Store failure_reason from Stripe
  ↓
  5. Send failure email:
     - Reason for decline
     - Support contact info
     - Retry instructions
  ↓
  6. Log email notification
  ↓
  7. Return 200 OK
  ↓
RESULT: User notified, payment logged, no credits added

================================================================================
IDEMPOTENCY PROTECTION
================================================================================

If webhook event received TWICE:
  ↓
  1. First time: stripe_session_id doesn't exist → Process normally
  ↓
  2. Second time: stripe_session_id EXISTS → Skip all processing
  ↓
  3. Return 200 OK immediately (same as first time)
  ↓
RESULT: Stripe sees 200 OK both times, stops retrying
         Credits added only ONCE (correct behavior)

MONITORING QUERIES
================================================================================

Recent successful payments:
  SELECT user_id, amount, invoice_url, created_at 
  FROM credit_transactions 
  WHERE payment_status = 'succeeded' 
  ORDER BY created_at DESC LIMIT 10;

Failed payments:
  SELECT user_id, failure_reason, created_at 
  FROM credit_transactions 
  WHERE payment_status = 'failed' 
  ORDER BY created_at DESC;

Webhook event status:
  SELECT event_type, status, error_message, processed_at 
  FROM stripe_webhook_logs 
  ORDER BY processed_at DESC LIMIT 20;

Email delivery status:
  SELECT notification_type, email_status, COUNT(*) 
  FROM payment_notifications 
  WHERE sent_at > NOW() - INTERVAL '7 days'
  GROUP BY notification_type, email_status;
